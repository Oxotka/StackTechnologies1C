# Руководство по внесению изменений

Спасибо, что хотите помочь сделать этот проект лучше!  
Любой вклад приветствуется: исправления ошибок, новые инструменты и технологии, улучшение формулировок, добавление примеров

## Как внести изменения

1. [Сделайте fork](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/working-with-forks/fork-a-repo) репозитория и склонируйте его локально
2. Внесите правки, следуйте стилю существующих разделов
3. Сделайте коммит с понятным сообщением:

```bash
git commit -m "Добавил пример интеграции с Telegram API"
```

4. Отправьте ветку в свой репозиторий:

```bash
git push origin main
```

5. Создайте pull request в мой репозиторий с описанием своих изменений

## Рекомендации по оформлению

- Пишите заголовки и описания чётко, без лишних слов
- Проверяйте орфографию и актуальность ссылок
- Примеры кода сопровождайте краткими пояснениями
- При добавлении нового инструмента указывайте:
  - Название и версию
  - Краткое описание (1–2 предложения)
  - Зачем нужен
  - Минимальный пример использования
  - Ссылки на официальную документацию

## Лицензия

Отправляя изменения, Вы соглашаетесь, что они будет распространяться по [лицензии MIT](LICENSE)
